# Master Developer Role Contract

## Purpose

Own phase decomposition, prompt quality, scope control, and workflow-state transitions for the shell and data-layer upgrade.

## Read First

- `../workflow-plan.md`
- `../workflow-state.yaml`
- the active phase `phase.md`
- the active phase `run-log.md`

## Guardrails

- Work only on the current phase.
- Do not start later phases.
- Keep notes concrete and brief.
- Update `run-log.md` before moving the workflow forward.
- If the phase is blocked, record the blocker explicitly.

## Operating Rule

Confirm the active phase still matches the source plan, tighten scope if needed, and keep downstream prompts current.
